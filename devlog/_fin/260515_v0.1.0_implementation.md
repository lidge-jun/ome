### 7. ChatGPT Thinking Code Review

Sent `ome-source.zip` (88KB) to ChatGPT thinking model via `agbrowse web-ai query`.

**Verdict: `NEEDS_FIX`** — good prototype, not production-ready.

| Area | Rating | Key Issue |
|------|--------|-----------|
| Architecture | WARN | Mutable globals exported, queue persistence never restored, incomplete features exposed |
| Type safety | WARN | `AgentCli` effectively `string`, DB/JSON boundary casts unvalidated |
| Error handling | **FAIL** | Unix process-tree kill broken (`detached: true` missing), timeout → zombie possible |
| Security | **FAIL** | Web API no auth, `0.0.0.0` bind exposes everything, raw errors leak |
| Test coverage | WARN | Happy paths only; no process-tree, auth, large-output tests |
| Performance | WARN | Log polling O(total), stdout unbounded memory accumulation |
| API design | WARN | Partial queue/session/quota exported as if complete |

**Top 3 priority fixes (from review):**
1. **Process supervision** — `detached: true` on Unix spawn, process group kill, "cancelling" state
2. **Web API auth** — token for `/api/*`, block non-loopback without auth, sanitize error messages
3. **Output/log bounding** — byte offset streaming, in-memory cap, stderr in SpawnResult

Review conversation: `https://chatgpt.com/c/6a06933e-b024-83a3-942a-352b4c5150fb`
